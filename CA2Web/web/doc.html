How we designed the system

There are 3 main components in our system; The server itself, a ServerClient that can send input and receive messages and our Clients.
When we run our program, a server are created on a given IP and port number, we initially used a simple local host.
When a user have successfully connected to the server, either through a GUI-Client or through a GUI-less client (TelNet) the server starts a new thread, and they get access to a ServerClient, where they are can login.
When they login, their ServerClient and name are put into a hashMap<String, ServerClient >, this is how we keep track of our users.

From here the user can type the commands the protocol allows eg. MSG:Mads:Hello.
The handler breaks the input into a 3 size string-array eg. “MSG” : “<receivers>” : “<message>”.
The handler identifies what command in the protocol has been called with the first string, and passes the receivers and the message to the server. The server uses the receivers to find the corresponding keys in our map, and sends the message to the clients.
When using a GUI-client we have filtered the out/input to be more user friendly.
